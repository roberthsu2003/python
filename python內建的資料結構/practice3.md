# 課堂範例回顧(學生剛才已練習)

## 原始程式

```python
# 建立一個英翻中對照字典
eng_to_zhtw = {
    "apple": "蘋果",
    "banana": "香蕉",
    "cherry": "櫻桃",
    "durian": "榴槤"
}

# 模擬快速輸入查詢
search_key = "banana"
# 使用 get 方法防錯，若查無此字便顯示提示
chinese_translation = eng_to_zhtw.get(search_key, "抱歉，字典中無此單字。")

print(f"單字 '{search_key}' 的中文翻譯是: {chinese_translation}")
```

## 使用的 Python 技術

- dictionary 初始化(大括號建立 key-value 對照表)
- 以 key 直接查詢對應的 value
- `dict.get(key, default)` 安全查詢,查無資料時回傳預設值,避免 `KeyError`
- 變數儲存查詢結果
- f-string 格式化輸出

## 學習目標

學生可以學會:

- 理解為什麼「大量資料的對照查詢」適合用 dictionary 而不是 list(O(1) vs O(n))
- 使用 key-value 的方式建立對照表
- 使用 `get()` 方法安全查詢,避免查無資料時程式當掉
- 體會 dictionary 在真實情境(條碼、學號、代碼查詢)中的應用價值

## 題目

小美發現顧客常常一次買好幾樣商品,收銀機需要一次掃描「多個條碼」並計算「總金額」。請你建立一個包含多個條碼的 list,例如 `["4710001", "4710002", "4710999"]`(其中 `"4710999"` 是不存在的條碼),使用迴圈搭配 `get()` 依序查詢每個條碼的價格並累加總金額;若查到不存在的條碼,該筆以 0 元計算,並額外印出「條碼 OOO 不存在,已略過」的提示。最後印出「總金額: OOO 元」。

## 提示

可以思考:

- 如何用 `for` 迴圈搭配 list,依序取出每個要查詢的條碼?
- `get()` 的預設值可以設成數字(例如 0)而不是文字訊息,這樣是不是比較方便直接拿去做加總?
- 如何在迴圈中判斷這筆條碼是「查到了」還是「用了預設值」,以便決定要不要印出「不存在」的提示?
