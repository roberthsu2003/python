# 實務應用四：個股日成交複雜資料清理

本單元將面對實務上最棘手的資料清理問題。我們將讀取「個股日成交資訊.csv」，其中包含許多非正規的格式：
1. **千分位逗號**：數值（如成交股數 `1,200,500`）包含逗號，無法直接被 Pydantic 轉換為整數 `int`。
2. **前後空白與修飾符號**：漲跌價差等浮點數欄位包含了多餘的空白以及符號（如 ` X 0.10 ` 或 ` +0.50 `）。

我們將學習如何利用 **BeforeValidator** 與 **Annotated** 來建構高度可重複使用的資料清洗型別。

## 1. 傳統字串模型 (無資料清洗)

如果不對欄位做任何型別轉換與清理，我們只能將所有數值宣告為 `str`。這會導致後續的數值計算（如加減乘除）極為不便，且程式內部充斥著手動轉型的代碼。

#轉換個股日成交資訊

import csv
from pprint import pprint

class Stack(BaseModel):
    代號: str = Field(alias='證券代號')
    名稱: str = Field(alias='證券名稱')
    成交股數: str
    成交金額: str
    開盤價: str
    最高價: str
    最低價: str
    收盤價: str
    漲跌價差: str
    成交筆數: str

with open('個股日成交資訊.csv', encoding='utf8', newline='') as csvfile:
    reader = csv.DictReader(csvfile)
    # 使用清單推導式進行批次驗證，取代 RootModel
    stocks = [Stack.model_validate(row) for row in reader]

for item in stocks:
    pprint(item)

## 2. 使用 BeforeValidator 與 Annotated 進行型別清洗

💡 **學習觀念 1：自訂清理函式**
我們首先定義了兩個底層清理函式：
* `_prepare_comma_seperated_int_`：負責將字串中的逗號 `,` 移除（如 `1,000` -> `1000`）。
* `_trim_float_`：負責去除字串兩側空白，並去除字首多餘的標記字元（如去除股票除權息時產生的 `X` 標記）。

💡 **學習觀念 2：定義高重用性的自訂型別 (Annotated + BeforeValidator)**
如果我們針對每個欄位都寫一個 `@field_validator(..., mode='before')`，模型代碼會變得很臃腫。
Pydantic 允許我們利用 Python 的 `typing.Annotated` 與 `BeforeValidator` 結合，定義成新型別：
* `CommaSeperatedInt = Annotated[int, BeforeValidator(_prepare_comma_seperated_int_)]`
* `CommaSeperatedFloat = Annotated[float, BeforeValidator(_prepare_comma_seperated_int_)]`
* `trimFloat = Annotated[float, BeforeValidator(_trim_float_)]`

隨後在 `Stack` 模型中，我們只需將欄位型別宣告為這些新型別，Pydantic 就會在資料填入前自動呼叫對應的清理函式，並安全地完成轉型。這極大地提升了代碼的優雅度與重用性！

```python
from pydantic import BaseModel, Field, BeforeValidator
from typing import Annotated
import csv
from pprint import pprint

def _prepare_comma_seperated_int_(value: str | int) -> str | int:
    if isinstance(value, str):
        return value.replace(",", "")
    return value

def _trim_float_(value: str) -> str:
    if isinstance(value, str):
        value = value.strip()
        value = value.lstrip('X')
        return value.strip()

CommaSeperatedInt = Annotated[int, BeforeValidator(_prepare_comma_seperated_int_)]
CommaSeperatedFloat = Annotated[float, BeforeValidator(_prepare_comma_seperated_int_)]
trimFloat = Annotated[float, BeforeValidator(_trim_float_)]

class Stack(BaseModel):
    代號: str = Field(alias='證券代號')
    名稱: str = Field(alias='證券名稱')
    成交股數: CommaSeperatedInt 
    成交金額: CommaSeperatedInt
    開盤價: CommaSeperatedFloat
    最高價: CommaSeperatedFloat
    最低價: CommaSeperatedFloat
    收盤價: CommaSeperatedFloat
    漲跌價差: trimFloat
    成交筆數: CommaSeperatedInt

with open('個股日成交資訊.csv', encoding='utf8', newline='') as csvfile:
    reader = csv.DictReader(csvfile)
    # 使用清單推導式進行批次驗證與資料清理
    stocks = [Stack.model_validate(row) for row in reader]

for item in stocks:
    pprint(item)

```

## 3. 取得特定股票的漲跌價差

💡 **學習觀念：欄位轉換驗證結果**
經過自訂型別 `trimFloat` 的轉換，原先的漲跌價差（例如包含 `X` 的除息資料）已被安全地解析為 Python 的 `float` 型別。由於 `stocks` 是一個標準 Python `list`，我們可以直接透過 `[index]` 來快速獲取特定個股的清洗結果。

## 4. FastAPI 實務整合與最佳設計

當我們需要將此個股日成交資料清洗邏輯整合至 FastAPI 專案中，且 **CSV 檔案已預先配置於伺服器端** 時，設計上會有以下調整：
1. **對外 API 欄位英文化**：輸入的 CSV 雖然是中文欄位（如 `證券代號`, `成交股數`），但 API 回傳的 JSON 應該是標準的英文 `snake_case`（如 `stock_id`, `trade_volume`）。
2. **複用自訂清洗型別**：在 Pydantic 模型中沿用 `Annotated` + `BeforeValidator`，在進入 Endpoint 時自動清洗千分位逗號以及多餘的 `X` 符號等髒資料，保證資料的高重用性與乾淨。
3. **避免 I/O 阻塞**：使用同步的 `def` 路由處理本地 Disk I/O，以利 FastAPI 自動透過外部執行緒池執行，防止主事件循環被阻塞。

### 💻 FastAPI 整合程式碼範例

```python
import csv
import os
from typing import Annotated
from fastapi import FastAPI, HTTPException, status
from pydantic import BaseModel, Field, BeforeValidator, ConfigDict

app = FastAPI(title="個股成交資訊清理系統")

# 預先配置在伺服器端的 CSV 檔案路徑
CSV_FILE_PATH = "個股日成交資訊.csv"

# 定義複用資料清洗函式
def _prepare_comma_seperated_int_(value: str | int) -> str | int:
    if isinstance(value, str):
        return value.replace(",", "")
    return value

def _trim_float_(value: str) -> str:
    if isinstance(value, str):
        value = value.strip()
        value = value.lstrip('X')
        return value.strip()

# 定義複用清洗型別
CommaSeperatedInt = Annotated[int, BeforeValidator(_prepare_comma_seperated_int_)]
CommaSeperatedFloat = Annotated[float, BeforeValidator(_prepare_comma_seperated_int_)]
trimFloat = Annotated[float, BeforeValidator(_trim_float_)]

# 定義單一股票資料模型 (API 輸出英文化)
class StockInfo(BaseModel):
    model_config = ConfigDict(populate_by_name=True)

    stock_id: str = Field(validation_alias="證券代號")
    name: str = Field(validation_alias="證券名稱")
    trade_volume: CommaSeperatedInt = Field(validation_alias="成交股數")
    trade_value: CommaSeperatedInt = Field(validation_alias="成交金額")
    open_price: CommaSeperatedFloat = Field(validation_alias="開盤價")
    high_price: CommaSeperatedFloat = Field(validation_alias="最高價")
    low_price: CommaSeperatedFloat = Field(validation_alias="最低價")
    close_price: CommaSeperatedFloat = Field(validation_alias="收盤價")
    price_change: trimFloat = Field(validation_alias="漲跌價差")
    transaction_count: CommaSeperatedInt = Field(validation_alias="成交筆數")

# API 回傳的標準包裹格式（Container Model）
class StockResponse(BaseModel):
    success: bool = True
    count: int
    stocks: list[StockInfo]

# 注意：此處使用 def 而非 async def，FastAPI 會自動在個別執行緒中執行，避免 Disk I/O 阻塞主事件循環
@app.get(
    "/stocks/records", 
    response_model=StockResponse,
    status_code=status.HTTP_200_OK,
    summary="讀取伺服器預置個股成交 CSV 檔案並清理資料"
)
def get_stock_records():
    # 檢查伺服器端檔案是否存在
    if not os.path.exists(CSV_FILE_PATH):
        raise HTTPException(
            status_code=status.HTTP_404_NOT_FOUND,
            detail=f"伺服器上找不到指定的 CSV 檔案：{CSV_FILE_PATH}"
        )
    
    try:
        stocks_list = []
        # utf-8-sig 能自動處理 Excel 產生的 BOM
        with open(CSV_FILE_PATH, encoding='utf-8-sig') as csvfile:
            reader = csv.DictReader(csvfile)
            for index, row in enumerate(reader, start=1):
                try:
                    # Pydantic 自動使用 BeforeValidator 清洗欄位並進行映射
                    stock = StockInfo.model_validate(row)
                    stocks_list.append(stock)
                except Exception as val_error:
                    raise HTTPException(
                        status_code=status.HTTP_422_UNPROCESSABLE_ENTITY,
                        detail=f"CSV 檔案第 {index} 行資料驗證清洗錯誤: {str(val_error)}"
                    )
        
        return StockResponse(
            count=len(stocks_list),
            stocks=stocks_list
        )

    except HTTPException as http_ex:
        raise http_ex
    except Exception as e:
        raise HTTPException(
            status_code=status.HTTP_500_INTERNAL_SERVER_ERROR,
            detail=f"伺服器讀取或解析 CSV 時發生錯誤: {str(e)}"
        )
```
